# Architecture Documentation Template

Every architecture diagram must be accompanied by a structured narrative section.

A diagram alone is not considered a complete deliverable.

## Standard Sections

| Section | Purpose |
|---|---|
| Objective | Why the architecture view exists |
| Scope | What is included and excluded |
| Primary Layer | Main layer represented by the diagram |
| Optional Overlays | Additional layers represented |
| Areas | Areas represented |
| Actors | Actors represented |
| Zones | Zones represented |
| Connectivity | Major connectivity patterns |
| Security Controls | Enforcement points and security services |
| Resilience | HA/DR characteristics |
| Assumptions | Design assumptions |
| Risks | Known risks or interpretation risks |
| Open Items | Items requiring validation or future refinement |
