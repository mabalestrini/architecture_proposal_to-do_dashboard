# Diagram Governance

## Core Rules

1. One diagram must tell one story.
2. Every diagram must declare a Primary Layer.
3. Optional overlays must be declared.
4. Areas and Zones must not be mixed.
5. Actors must be shown as actors, not infrastructure areas.
6. Boundary crossings must be explicit in L2+ diagrams.
7. Security controls must sit on or near enforcement boundaries.
8. Administrative flows must be visually distinct.
9. Temporary or exception flows must be dashed and dated.
10. Diagrams must remain readable in grayscale.

## Required Flow Labels

When flows are shown, labels should include:

```text
Protocol / Port / Purpose
```

For security-sensitive flows, include encryption or authentication where relevant.

## Review Checklist

- [ ] Diagram title is clear.
- [ ] Primary Layer is stated.
- [ ] Overlays are stated.
- [ ] Scope is stated.
- [ ] Areas are clearly identified.
- [ ] Zones are clearly identified where relevant.
- [ ] Actors are clearly identified.
- [ ] Boundary crossings are explicit where relevant.
- [ ] Security controls are shown where relevant.
- [ ] Legend is included if colors or line styles matter.
- [ ] Diagram is readable in grayscale.
